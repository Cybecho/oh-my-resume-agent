# data/ — legacy private-data location

This directory is intentionally empty in the public repository.

The seed project used `data/` for personal profiles, experience cards,
claim registries, writing samples, and resume snapshots. Those files are
private user evidence and must not be committed to the public template.

For new installs, place raw inputs in `userinfo/` and let the agent normalize
public-safe working artifacts into `workspace/`.
