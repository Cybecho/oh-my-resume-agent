# 이어서 — 세션 복원

이전 세션에서 중단된 자소서 작업을 이어서 진행한다.

## 실행

1. `output/` 폴더를 스캔하여 모든 회사 폴더 목록을 확인한다
2. 각 폴더의 `state.json`을 읽어 현재 상태를 파악한다
3. 사용자에게 현황을 표시한다:

```
═══ 자소서 작업 현황 ═══

1. 삼성전자_20260212 — 전체 완료 ✓
2. 카카오_20260215 — Step 3까지 완료 (다음: /매칭)
3. 네이버_20260218 — Step 1 진행중
```

4. 미완료 작업이 있으면:
   - "이어서 진행할 작업을 선택하세요" 안내
   - 사용자가 선택하면 해당 작업의 다음 단계부터 자동 실행
   - (마스터 스킬 `/자소서`의 해당 단계부터 이어서 진행하는 방식)

5. 모든 작업이 완료 상태면:
   - "진행중인 작업이 없습니다. `/자소서`로 새 작업을 시작하세요."

## 이어서 진행 방식

- state.json의 `current_step`을 확인하여 해당 단계의 에이전트 지침부터 실행
- 이전 단계 output 파일들이 모두 존재하는지 확인
- 누락된 파일이 있으면 해당 단계부터 재실행 제안

## 특수 케이스

- `state.json`이 없거나 손상된 폴더: "상태 파일이 손상되었습니다" 표시
- `in_progress` 상태에서 중단된 경우: 해당 단계를 처음부터 재실행
