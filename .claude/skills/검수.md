# 검수 — 검수 단독 재실행

완성된 자소서 3세트에 대한 검수를 단독으로 재실행한다.

## 사전 조건

1. `output/` 폴더에서 대상 회사 폴더를 확인한다
   - 여러 개면 사용자에게 선택 요청
2. 해당 폴더의 `state.json`을 읽는다
3. `07_자소서/컨셉{1,2,3}.md`가 존재하는지 확인

## 실행

1. `agents/검수선생.md`를 Read로 읽는다
2. 전 단계 output 전부 읽기 (state.json ~ 07_자소서/)
3. `data/experience_cards/`에서 경험 원문 우선 검증하고, 필요 시 `data/experiences/` 및 Notion으로 보완
4. `data/writing_samples/` 읽기 (문체 비교용)
5. 8항목 평가 수행:
   - JD 적합도, 인재상 부합도, 논리적 일관성
   - 기업문화 어휘 반영도, 원천데이터 팩트체크
   - 문체 자연스러움, 글자수 검증
   - 3세트 비교 + 추천 순위
6. `08_검수리포트.md` 덮어쓰기 저장
7. `state.json` 업데이트: review → completed

## 완료 후

- 추천 순위 표시 (1위 컨셉 + 이유)
- 즉시 수정 필요 사항 표시 (있는 경우)
- "피드백을 등록하려면 `/피드백`을 사용하세요."
