# 딸깍자소서 선생 — 전체 파이프라인 자동 실행

사용자로부터 회사명, JD, 자소서 문항을 입력받아 7단계를 자동으로 순차 실행하여 3세트의 자기소개서를 작성한다.

## 입력 받기

사용자에게 아래 정보를 요청한다:
- **회사명**: 지원 회사 정식명칭
- **지원 직무**: 직무명
- **JD**: 채용공고 전문 텍스트 (담당업무, 자격요건, 우대사항 등)
- **자소서 문항**: 각 문항 텍스트 + 글자수 제한

모든 입력이 확보되면 파이프라인을 시작한다.

## 초기화

1. 오늘 날짜를 YYYYMMDD 형식으로 구한다
2. `output/{YYYYMMDD}_{회사명}/` 폴더를 생성한다
3. `output/{YYYYMMDD}_{회사명}/07_자소서/` 하위 폴더도 생성한다
4. `state.json`을 아래 형식으로 생성한다:
```json
{
  "company": "{회사명}",
  "position": "{직무}",
  "created_at": "{현재시각 ISO8601}",
  "updated_at": "{현재시각}",
  "current_step": 1,
  "jd_text": "{JD 전문}",
  "questions": [
    { "id": 1, "text": "{문항1}", "char_limit": {글자수} }
  ],
  "steps": {
    "1": { "status": "pending", "file": "01_기업조사.md" },
    "2": { "status": "pending", "file": "02_JD분석.md" },
    "3": { "status": "pending", "file": "03_의도분석.md" },
    "4": { "status": "pending", "file": "04_경험풀.md" },
    "5": { "status": "pending", "file": "05_경험매칭.md" },
    "6": { "status": "pending", "file": "06_기획.md" },
    "7": { "status": "pending", "file": null },
    "review": { "status": "pending", "file": "08_검수리포트.md" }
  }
}
```

## 파이프라인 실행

각 단계에서: (1) 해당 에이전트 지침을 Read로 읽고, (2) 이전 단계 output을 읽고, (3) 지침대로 수행하고, (4) output 파일 저장, (5) state.json 업데이트.

진행상황을 사용자에게 표시한다:

---

### Step 1: 기업 조사

```
── Step 1/7: 기업 조사 ──
```

1. `agents/조사선생.md`를 Read로 읽는다
2. 지침에 따라 WebSearch/WebFetch로 기업정보 조사 실행
3. Culture-Sync 어휘 추출 포함
4. 결과를 `output/{YYYYMMDD}_{회사명}/01_기업조사.md`에 저장 (`templates/schemas/company_profile.md` 형식)
5. state.json: step 1 → completed, current_step → 2

```
✓ 01_기업조사.md 저장
```

---

### Step 2-3: JD 분석 + 의도 추론

```
── Step 2-3/7: JD 분석 + 의도 추론 ──
```

1. `agents/JD분석선생.md`를 Read로 읽는다
2. state.json에서 JD 텍스트, 01_기업조사.md 읽기
3. JD 구조화 → 필수/우대 역량 분류 → 실제 업무 추론
4. `output/{YYYYMMDD}_{회사명}/02_JD분석.md`에 저장

5. `agents/인사관선생.md`를 Read로 읽는다
6. 01_기업조사.md + 02_JD분석.md + 문항 목록 참조
7. 채용 배경 추론 → 문항별 출제 의도 분석
8. `output/{YYYYMMDD}_{회사명}/03_의도분석.md`에 저장
9. state.json: step 2,3 → completed, current_step → 4

```
✓ 02_JD분석.md 저장
✓ 03_의도분석.md 저장
```

---

### Step 4-5: 경험 매칭

```
── Step 4-5/7: 경험 매칭 ──
```

1. `agents/기억선생.md`를 Read로 읽는다
2. 02_JD분석.md, 03_의도분석.md, 01_기업조사.md 읽기
3. 로컬 `data/experience_cards/`에서 경험카드 기반 후보군 선별
4. `data/experiences/_index.md` 및 관련 파일 읽기 (legacy 보완)
5. `data/feedback/log.json` 읽기 (피드백 가중치)
6. 1차 인덱싱 → `output/{YYYYMMDD}_{회사명}/04_경험풀.md` 저장
7. 2차 인덱싱 → `output/{YYYYMMDD}_{회사명}/05_경험매칭.md` 저장
8. state.json: step 4,5 → completed, current_step → 6

```
✓ 04_경험풀.md 저장
✓ 05_경험매칭.md 저장
```

---

### Step 6: 기획

```
── Step 6/7: 기획 ──
```

1. `agents/기획선생.md`를 Read로 읽는다
2. 01_기업조사.md(Culture-Sync), 03_의도분석.md, 05_경험매칭.md 읽기
3. 3가지 컨셉 설계 (전체 아크 + 항목별 경험 배치)
4. `output/{YYYYMMDD}_{회사명}/06_기획.md`에 저장
5. state.json: step 6 → completed, current_step → 7

```
✓ 06_기획.md 저장
```

3가지 컨셉 요약을 사용자에게 표시한다.

---

### Step 7: 작성

```
── Step 7/7: 자소서 작성 ──
```

1. `agents/필체선생.md`를 Read로 읽는다
2. 06_기획.md, 05_경험매칭.md 읽기
3. `data/writing_samples/` 전체 읽기 (문체 학습)
4. 경험 원문 읽기 (`data/experience_cards/` 및 `data/experiences/`, Notion)
5. state.json에서 문항 + 글자수 확인
6. 3세트 자소서 작성
7. 저장:
   - `output/{YYYYMMDD}_{회사명}/07_자소서/컨셉1.md`
   - `output/{YYYYMMDD}_{회사명}/07_자소서/컨셉2.md`
   - `output/{YYYYMMDD}_{회사명}/07_자소서/컨셉3.md`
8. state.json: step 7 → completed

```
✓ 07_자소서/컨셉1.md 저장
✓ 07_자소서/컨셉2.md 저장
✓ 07_자소서/컨셉3.md 저장
```

── 검수 ──
