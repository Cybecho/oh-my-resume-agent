# 피드백 — 결과 피드백 수집

자소서 제출 후 결과 피드백을 수집하여 향후 자소서 작성에 반영한다.

## 실행

### 1. 대상 확인
- `output/` 폴더를 스캔하여 완료된 작업 목록 표시
- 사용자에게 "어떤 자소서에 대한 피드백인가요?" 질문

### 2. 피드백 수집
사용자에게 아래 정보를 요청한다:

- **결과**: 서류통과 / 탈락 / 면접통과 / 최종합격 / 기타
- **사용한 컨셉**: 컨셉1 / 컨셉2 / 컨셉3
- **효과적이었던 점** (선택): "이 표현이 좋았다", "이 경험이 어필했다" 등
- **아쉬운 점** (선택): "이 부분을 고쳤으면", "이 경험은 안 맞았다" 등
- **메모** (선택): 면접 피드백, 추가 소감 등

### 3. 피드백 저장
`workspace/feedback/log.json`을 읽고, 새 피드백을 추가한다. 기존 개인용 seed에서만 `data/feedback/log.json`를 legacy fallback으로 사용한다:

```json
{
  "date": "{오늘 날짜}",
  "company": "{회사명}",
  "position": "{직무}",
  "result": "{결과}",
  "concept_used": "{사용 컨셉}",
  "effective_experiences": ["{경험명1}", "{경험명2}"],
  "effective_expressions": ["{효과적 표현1}"],
  "weak_points": ["{아쉬운점1}"],
  "notes": "{메모}"
}
```

### 4. 가중치 반영 안내
- 서류통과/면접통과/최종합격: 해당 경험의 가중치가 높아짐
- 효과적이었던 표현: 필체선생이 향후 참조
- 아쉬운 점: 기획선생/필체선생이 향후 회피

피드백이 저장되면:
```
✓ 피드백 저장 완료 (workspace/feedback/log.json)
다음 자소서 작성 시 /매칭 단계에서 이 피드백이 반영됩니다.
```

## 피드백 현황 조회

사용자가 "피드백 현황"을 요청하면:
- `workspace/feedback/log.json` 전체 읽기
- 기존 개인용 seed에서만 `data/feedback/log.json`를 legacy fallback으로 사용
- 통과율, 자주 사용된 경험, 효과적이었던 표현 등 통계 표시
